How to Contribute

Never used GitHub? No problem! Follow this beginner-friendly guide to contribute to the Cozy Manifesto.
Jump to the steps

Welcome! The Cozy Manifesto is a community-driven project, and your contributions can help make it even better. Whether it’s fixing a typo, adding new insights, or sharing your perspective, anyone can contribute—even if you’ve never used GitHub before.

This guide will show you how to contribute directly in your web browser without any special tools or coding knowledge.

STEP-BY-STEP GUIDE

How to Contribute

Step 1: Access the GitHub Page

Open your web browser and navigate to the Cozy Manifesto GitHub page. The URL should look something like this:
https://github.com/username/cozy-manifesto

Step 2: Sign In or Create a GitHub Account

  • If you already have a GitHub account, click Sign in in the top-right corner of the page.
  • If you don’t have an account:
    • Click Sign up in the top-right corner.
    • Follow the instructions to create a free account.

Step 3: Fork the Repository

To make changes, you first need to create your own copy of the repository:

  1. On the Cozy Manifesto repository page, click the Fork button at the top-right corner.
  2. This will create a copy of the repository in your own GitHub account.

Step 4: Edit the Manifesto

  1. In your forked repository, find the file you want to edit (e.g., manifesto.md).
  2. Click on the file name to open it.
  3. Click the Edit button (a pencil icon) in the top-right corner of the file viewer.
  4. Make your changes directly in the text editor that appears.

Step 5: Preview Your Changes

Scroll down to the Commit Changes section at the bottom of the page:

  1. Click Preview changes to see what your edits will look like.
  2. Review your changes carefully.

Step 6: Propose Changes (Pull Request)

Once you’re satisfied with your edits:

  1. In the Commit changes section, enter a short description of your changes (e.g., "Improved wording in Chapter 2").
  2. Select Create a new branch for this commit and start a pull request.
  3. Click Propose changes.

Step 7: Create a Pull Request

  1. After clicking Propose changes, you’ll be taken to a page to create a Pull Request.
  2. Add a title and a brief description of your contribution.
  3. Click Create Pull Request.

Step 8: Wait for Approval

Your Pull Request will be reviewed by the repository owner. You may receive feedback or questions, so check back for updates!
